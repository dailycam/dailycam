# Gemini 분석 결과 JSON → TXT 변환 사용법

## 기능
- Gemini API 분석 결과 JSON을 사람이 읽기 쉬운 TXT 파일로 자동 변환
- 원본 JSON도 함께 저장 (디버깅용)

## 자동 저장 위치
```
backend/analysis_results/
├── analysis_stage3_20251127_130500.txt     # 포맷팅된 분석 결과
├── raw_stage3_final_20251127_130500.json   # 원본 JSON
└── ...
```

## 사용 방법

### 1. API로 비디오 분석 시 자동 저장
비디오를 업로드하면 자동으로 TXT와 JSON 파일이 생성됩니다.

```bash
# 비디오 분석 API 호출 (자동으로 파일 저장됨)
curl -X POST "http://localhost:8000/api/homecam/analyze-video" \
  -F "video=@test_video.mp4" \
  -F "age_months=7"
```

### 2. 코드에서 직접 사용

```python
from app.utils.json_to_txt_formatter import GeminiAnalysisFormatter

# Formatter 인스턴스 생성
formatter = GeminiAnalysisFormatter(output_dir="my_results")

# Gemini 분석 결과 JSON
analysis_result = {
    "meta": {
        "assumed_stage": "3",
        "age_months": 7,
        ...
    },
    "development_analysis": { ... },
    "safety_analysis": { ... }
}

# TXT 파일로 저장
txt_path = formatter.format_and_save(
    analysis_data=analysis_result,
    filename_prefix="test_analysis"
)
print(f"저장 완료: {txt_path}")

# 원본 JSON도 저장
json_path = formatter.save_raw_json(
    data=analysis_result,
    stage="final",
    filename_prefix="test_raw"
)
```

## 테스트 방법

### 1. 비디오 분석 후 파일 확인
```bash
# 1. 비디오 분석 실행
python -m uvicorn app.main:app --reload

# 2. API 호출
curl -X POST "http://localhost:8000/api/homecam/analyze-video" \
  -F "video=@test_video.mp4"

# 3. 결과 파일 확인
ls -la backend/analysis_results/
```

### 2. 터미널에서 확인
분석이 완료되면 터미널에 다음과 같이 출력됩니다:
```

✅ 분석 결과 TXT 파일 저장 완료: analysis_results/analysis_stage3_20251127_130500.txt
📄 TXT 파일 저장 완료: analysis_results/analysis_stage3_20251127_130500.txt
✅ 원본 JSON 저장 완료: analysis_results/raw_stage3_final_20251127_130500.json
📄 원본 JSON 저장 완료: analysis_results/raw_stage3_final_20251127_130500.json
```

## 파일 구조

### TXT 파일 내용 구성
1. **메타 정보**: 발달 단계, 개월 수, 관찰 시간 등
2. **발달 단계 판단**: 판단된 단계, 신뢰도, 근거
3. **발달 분석**: 관찰된 기술, 단계 일치도, 다음 단계 징후
4. **안전 분석**: 안전 점수, 사고 이벤트, 환경 위험 요소
5. **환경 정보**: 장소, 가구, 안전장치
6. **보호자 상호작용**: 동반 빈도, 상호작용 기록

### JSON 파일
- 원본 Gemini 응답 그대로 저장 (indent=2로 포맷팅)

## 주의사항
- 파일은 `backend/analysis_results/` 디렉토리에 자동 생성됩니다
- 파일명에 타임스탬프가 포함되어 덮어쓰기 걱정 없습니다
- TXT 파일은 UTF-8 인코딩으로 저장됩니다
